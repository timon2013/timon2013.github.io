New Console-to-Code Service for Amazon AWS

October 13, 2024 · 2 min aby przeczytać
Piotr Kmita
Maintainer of site

AWS has introduced the Console-to-Code service, which allows users to easily transform actions performed in the AWS console into code, significantly simplifying the automation and application deployment process. Here’s how this service works and how you can use it.

How does Console-to-Code work?​

  1. Action Recording: Console-to-Code automatically records all user actions in the AWS console, such as creating an EC2 instance or configuring a VPC. The user performs standard operations in the console, and the system records these actions.

  2. Code Generation: After completing the actions, the user can go to the Console-to-Code panel, where they can select specific actions to be transformed into code. The service generates the code in the preferred format, such as:

    • AWS CloudFormation (JSON or YAML)
    • AWS CDK (Java, Python, TypeScript)

  3. Code Customization: The generated code can be used as a starting point for further customization and adjustment to specific production needs. Users are encouraged to test and verify the generated code before deploying it.

How to use Console-to-Code?​

Step by step:​

  1. Log into the AWS console: Make sure you are in the US East (N. Virginia) region, as the service is currently only available in this region.

  2. Select the EC2 service: Start the process, for example, by launching an EC2 instance.

  3. Open the Console-to-Code panel: After performing the actions, find the new "Console-to-Code" option in the left panel.

  4. Select actions to transform: Check the actions you want to include in the generated code (up to 5 actions at a time).

  5. Generate code: Click the button to generate code and choose the preferred format.

  6. Download or copy the code: After generating, you can copy or download the code for further editing.

Limitations​

  • The service is currently only available for actions related to Amazon EC2, Amazon VPC, and Amazon RDS.
  • The ability to record actions is limited to five actions per session.
  • The generated code requires further customization before being used in production.

Summary​

AWS Console-to-Code is an innovative tool that combines the simplicity of using the console with the power of Infrastructure-as-Code (IaC). It enables users to easily transition from prototyping to production, making it a valuable addition for developers and DevOps engineers.
